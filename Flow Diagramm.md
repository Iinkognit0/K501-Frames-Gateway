# Flow-Diagramm  

## Trigger → Träger → Archiv-Ablauf  

*(Iinkognit0 · Das ewige Archiv)*

  

---

  

## 1. Auslöser (Trigger)

[ Trigger erkannt ]

Beispiele:

- Projektanalyse  

- Projektprognose  

- Projektstatistiken  

- Output MD  

- Struktur-Änderung  

  

Eigenschaften:

- kurzlebig  

- nicht speichernd  

- initiierend  

  

↓

  

---

  

## 2. Vorab-Prüfung (Validator)

[ Validator-Checkliste ]

Prüft:

- Ist-Zustand aktuell?

- Träger betroffen?

- Konflikt mit Grundsätzen?

- Priorität korrekt?

- Archiv-Referenz verletzt?

  

Ergebnis:

- ❌ Abbruch → Hinweis, keine Änderung  

- ✔ Weiter  

  

↓

  

---

  

## 3. Prioritätenlogik (Trigger)

[ Trigger-Priorität ]

Reihenfolge:

1. Ordnung / Struktur  

2. Zustand / Fakten  

3. Prognose / Statistik  

4. Inhalt  

5. Erläuterung  

  

Regel:

Niedrigere Trigger warten, wenn höhere aktiv sind.

  

↓

  

---

  

## 4. Träger-Entscheidung

[ Betroffener Träger ]

Mögliche Träger:

- Website (Ursprung)  

- Obsidian Vault (Gedächtnis)  

- Podcast (Inhalt)  

- Essay / Manifesto (Verdichtung)  

- Stimme Iinkognit0 (Meta)  

  

Grundsatz:

Trigger verändern Träger **nie direkt**,  

sondern nur über geprüfte Ablage.

  

↓

  

---

  

## 5. Ausgabeform bestimmen

[ Ausgabeformat ]

- MD → Archiv / Gedächtnis  

- YAML → Referenz / Logik  

- SQL → Persistenz / Struktur  

- Text → temporär / Denkraum  

  

↓

  

---

  

## 6. Ablage im Archiv

[ Archivierung ]

- korrekter Ordner  

- korrekter Namensstandard  

- Status setzen:

  - ✔ abgeschlossen  

  - archivfähig  

  - kanonisch (falls zutreffend)  

  

↓

  

---

  

## 7. Rückkopplung (optional)

[ Rückmeldung ]

- Projektstatus aktualisieren  

- Meta-Ebene anpassen  

- keine Wiederholung notwendig  

  

↓

  

---

  

## 8. Ruhezustand

[ System stabil ]

- kein offener Trigger  

- keine Aktion  

- Archiv trägt den Zustand  

  

---

  

## Kurzform (Essenz)

Trigger

→ Validator

→ Priorität

→ Träger

→ Ausgabe

→ Archiv

→ Ruhe

---

  

## Schluss

  

Dieses Flow-Diagramm beschreibt keinen Prozess,

sondern einen Zustandsübergang.

  

Es erzwingt nichts.  

Es schützt Ordnung.  

Es bewahrt Dauer.
