Quantum Archivator Language (QAL)


Chapter 5 ‚Äî Index Layers and Presence Proofs (L0 / L1)


This chapter defines how existence and structure are made observable without interpreting content.

5.1 Index Layers Overview


QAL distinguishes content-bearing Frames from indexing Frames.

Indexing Frames never add semantics; they only report presence and structure.


Two canonical layers are defined:

  • L0 ‚Äî Presence Index

  • L1 ‚Äî Verified Index

5.2 L0: Presence Index


The L0 Index is a pointer-only inventory.


It lists:

  • Frame identifiers,

  • basic metadata (type, byte size),

  • ordering position.


Formally, let \mathcal{F} = \{F_1,\dots,F_n\}.

Then:

\text{L0}(\mathcal{F}) = \{\, id(F_i) \mid 1 \le i \le n \,\}


Properties:

  • no hashes required,

  • no validation performed,

  • existence claim only.


L0 answers the question:

‚ÄúWhat exists?‚Äù

5.3 L1: Verified Index


The L1 Index extends L0 with cryptographic verification.


It includes:

  • self-hash of each Frame,

  • previous-hash linkage,

  • optional pack checksum references.


Formally:

\text{L1}(\mathcal{F}) = \{\, (id(F_i), h_{self}(F_i), h_{prev}(F_i)) \,\}


L1 answers the question:

‚ÄúWhat exists, and can its structure be verified?‚Äù

5.4 Presence Proof vs. Validity Proof


A critical distinction:

  • Presence Proof: the Frame exists in the archive.

  • Validity Proof: the Frame‚Äôs structure is consistent.


QAL guarantees presence proofs by L0

and structural validity proofs by L1.


No index asserts truth, correctness, or meaning.

5.5 Monotonic Index Growth


Indices are append-only.


Let I_k be an index over \mathcal{F}_k.

After appending F_{k+1}:

I_{k+1} = I_k \cup \{F_{k+1}\}


Previous index states remain reproducible.

5.6 Offline and Tool-Agnostic Verification


Index verification requires only:

  • the index file,

  • the referenced Frames,

  • a standard hash function.


No runtime, no execution, no external service.
